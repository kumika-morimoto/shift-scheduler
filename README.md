# シフト生成アプリ

Google Gemini で生成したコードをベースにした、React + Vite + Tailwind のシフト自動生成アプリ


## プロジェクト概要

本アプリは、スタッフの希望時間・勤務タイプ（free / fixed）・週の勤務日数・希望時間帯などの条件をもとに、シフトを自動生成する **シフト生成アプリ** です。

UI とロジックの初期生成には、Google Gemini（無料プラン / Canvas 高速モード）を使用し、**コードを書かず、言語によるプロンプトのみでアプリ構造を生成しました。**

生成されたコードをもとに、React + Vite + Tailwind CSS でローカル環境向けに再構築しています。

現在はプロトタイプ段階で、今後は UI/UX の改善、コンポーネント分割、シフト生成ロジックの拡張などを予定しています。


## 主な機能

- スタッフの新規追加・削除
- スタッフごとの勤務タイプ（free / fixed）の設定
- 1日の最大勤務時間・週の勤務日数の設定
- 希望時間帯（freeHours）の登録
- 曜日ごとの希望時間（pref）の設定
- 休み希望日の登録（offDates）
- 条件に基づくシフトの自動生成
- スタッフごとの色分け表示
- 生成されたシフト表の確認


## 設計上の前提条件

本アプリのシフト生成ロジックは、以下の前提条件をもとに設計しています。

- 24時間交代制で、基本は **日中帯：2人体制 / 夜間帯：1人体制**
- 夜間帯は必ず登録スタッフが入る（スポットバイトでは埋めない）
- 空席（スポットバイトで埋める枠）は **日中帯にのみ発生する**
- スタッフが **ぶつ切れにならず、連続した時間で勤務できるように** シフトを生成
- 各スタッフの **希望最大勤務時間にできるだけ近づけるように** 調整
- 本アプリは「完全なシフト表を自動生成する」ことを目的とせず、 **参考値として利用する前提で、あえて余白を残す設計**

これらの前提に基づき、スタッフの希望条件や勤務タイプを考慮しながらシフトの自動生成を行います。


## 使用技術

- **React 18**  
  UI を構築するためのライブラリ

- **Vite 5**  
  開発サーバー & ビルドツール  
  （高速で、React プロジェクトの標準的な選択）

- **JavaScript (ES Modules)**  
  プロジェクト全体の言語

- **Tailwind CSS 3.4.4（安定版）**  
  UI スタイルを記述するためのユーティリティファースト CSS  
  ※ v4 は仕様が大きく変わるため、今回は安定版の v3 を採用

- **lucide-react**  
  UI アイコンライブラリ  
  （Gemini のコードで使用されているため導入）

- **Node.js / npm**  
  パッケージ管理と開発環境の実行に使用

- **Google Gemini（コード生成に使用）**
  UI とロジックの初期生成を、言語プロンプトのみで生成


## セットアップ手順（ローカル環境）

### 1. リポジトリをクローン
```bash
git clone https://github.com/kumika-morimoto/shift-scheduler.git
cd shift-scheduler
```

### 2. 依存パッケージをインストール
```bash
npm install
```

### 3. 開発サーバーを起動
```bash
npm run dev
```

ブラウザで表示される URL（例: http://localhost:5173）にアクセスするとアプリが動作します。


### アプリコードについて

本プロジェクトでは、Google Gemini（無料プラン / Canvas 高速モード）で生成したアプリケーションコードを `src/App.jsx` に配置しています。

React プロジェクトでは `App.jsx` がメインコンポーネントとして機能するため、UI やロジックはこのファイルにまとめています。

本リポジトリには、生成済みの App.jsx が含まれているため、そのまま動作します。
